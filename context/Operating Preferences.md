# Operating Preferences

_Generated by `/onboard`. Update directly when you notice a pattern that worked or failed._

## Communication
- Response length: terse by default; verbose only when asked.
- Trailing summaries: avoid.
- Status narration: brief one-line updates between actions, not paragraphs.
- Banned words / slop: see `CLAUDE.md` Core Rules #10.

## Engineering
- Test-first when feasible (RED → GREEN → REFACTOR).
- Root-cause before patch — no symptom hiding.
- Small commits with structured trailers (`Constraint:`, `Rejected:`, `Confidence:`, `Scope-risk:`, `Not-tested:`) for non-trivial work.
- Prefer existing repo patterns over introducing new abstractions.

## Approval defaults
- T0 (safe / local) — execute immediately.
- T1 (local writes) — log and proceed.
- T2 (shared state, deploys, public actions) — wait for explicit approval.
- T3 (destructive, irreversible) — refuse unless pre-authorized.

## Tools and platforms
- Terminal: Warp (cockpit only — no governance there).
- Execution engine: Claude Code with this repo's hooks active.
- Code intelligence: SocratiCode preferred over broad grep/Read for non-trivial exploration.
- Memory: file-based `memory/` + claude-mem semantic + `kb/wiki/` curated.
